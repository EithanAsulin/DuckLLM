Common Questions

Q: Is DuckLLM truly private and local?

A: Yes. Because it utilizes Ollama as the backend, the Model Runs On Your Machine Without Any Cloud Based Requirements.

Q: Why do I need Python installed separately?

A: For Multiple Operating Systems, DuckLLM Requires a Cross-Platform Compatible GUI Engine.

Q: Can I use other models besides Qwen 2.5 Vision?

A: Yes, By Editing The Model File Before Installing You Can Use a Different Ollama Model. Do Beaware You Might Experience Issues.

Q: What If i Don't Have a Discrete GPU?

A: Ollama will automatically fall back to CPU inference if no discrete GPU is detected. While slower, this ensures compatibility with all systems.

Q: How do I update to the latest version?

A: Simply check the "Releases" section on GitHub. You can download the latest installer and run it over your existing installation.

Q: Wouldn't Using a Vision Model Cause Performance Issues Compared To Other Models?

A: No, Only Once an Image Is Selected It Will Use DuckLLM's Vision Engine (Dual Mode).

Q: Why Does The Model File Include The Country It Was Developed In? (Israel)

A: This Is Just a Placeholder Since If Not Added The Entire Model Identity Might Be Lost Or It Might Keep Mentioning Other Unrelated Countries. (Could Be Expected To Remove In V2.5.0)
